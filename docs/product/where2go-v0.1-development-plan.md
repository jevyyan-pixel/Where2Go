# Where2Go V0.1 开发计划

## 已确认决策

- 最低系统版本：iOS 17+
- 技术栈：SwiftUI + SwiftData + MVVM + Services
- 地图：V0.1 暂不开发
- 通知：保留每日行程摘要，并支持后续单事项提醒
- 工程：创建真实 Xcode 工程

## 阶段 0：项目初始化与骨架

目标：搭建一个可打开、可继续开发的 iOS 工程。

已完成：

- 创建 `Where2Go.xcodeproj`
- 创建 SwiftUI App 入口
- 接入 SwiftData model container
- 建立 4 个底部 Tab：下一程、日历、时间线、设置
- 建立基础目录：Models、Views、Services、DesignSystem、Resources、Tests

## 阶段 1：行程数据闭环

目标：跑通“新增行程 -> 本地保存 -> 页面展示”的第一条主链路。

已完成：

- `TripItem` SwiftData 模型
- `TripCategory` 类型枚举
- `ReminderOption` 提醒枚举
- `TripFormView` 新增行程表单
- `TripQueryService` 日期筛选、分组和摘要逻辑
- 时间线页面按日期展示行程
- 日历页面显示当月日期和有安排标记
- 下一程页面显示最近行程摘要

下一步：

- 补充更多单元测试

已继续完成：

- 增加 `TripDetailView` 行程详情页
- `TripFormView` 支持新增/编辑共用
- 下一程、日历、时间线的行程行可进入详情
- 删除操作统一放到详情页确认弹窗
- 补充最近行程与日期分组测试用例

下一步：

- 通知摘要内容接入真实当天行程
- 删除后提供撤销能力
- 增加更多 UI 状态与可访问性检查

## 阶段 2：行程摘要与通知

目标：让用户每天能看到或收到行程摘要。

已完成：

- 设置页增加“每日行程摘要”开关
- `NotificationService` 预留每日摘要通知接口
- 摘要通知内容从真实当天行程生成
- App 启动、行程变化、摘要设置变化后自动重排未来 7 天摘要
- 设置摘要时间后重新调度通知

下一步：

- 增加通知权限状态展示
- 增加通知失败/无权限引导
- 在设置页展示下一次摘要时间

## 阶段 3：产品打磨

目标：让 V0.1 具备可试用质量。

待做：

- 空状态文案统一
- 深色模式检查
- 动态字体检查
- VoiceOver 标签补全
- 首页摘要文案优化
- 日历滚动和跨月体验优化
- 设置页数据管理入口

已继续完成：

- 外观设置应用到全 App
- 设置页显示通知权限、下一次摘要时间、已排摘要数量
- 设置页支持 JSON 备份导出
- 设置页支持清空本地全部行程
- 增加 V0.1 App 图标资源
- 增加项目 README

## 暂缓到 V0.2+

- 地图预览与地点搜索
- 系统日历 EventKit 同步
- iCloud / CloudKit 多设备同步
- 行程分享
- 自定义行程类型
